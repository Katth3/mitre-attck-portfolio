# Lab Report — Atomic Red Team T1059.001 PowerShell Execution

**Author:** Edna Katerine Ruiz Gutierrez
**Date:** June 2026  
**Objective:** Emulate T1059.001 PowerShell execution techniques using
Atomic Red Team and validate detection coverage in Splunk.  
**Technique:** T1059.001 — Command and Scripting Interpreter: PowerShell  
**Tactic:** Execution

---

## Environment

| Component | Details |
|-----------|---------|
| Victim VM | Windows 10 Enterprise + Sysmon v15 |
| Emulation tool | Atomic Red Team v4 (Invoke-AtomicRedTeam) |
| SIEM | Splunk Enterprise 9.x |
| Log sources | Sysmon EID 1, Windows Security EID 4104 |

---

## Atomic Red Team Installation

Run in PowerShell as Administrator on the victim VM:

```powershell
IEX (IWR 'https://raw.githubusercontent.com/redcanaryco/invoke-atomicredteam/master/install-atomicredteam.ps1' -UseBasicParsing)
Install-AtomicRedTeam -getAtomics
```

Verify installation:

```powershell
Import-Module invoke-atomicredteam
Invoke-AtomicTest T1059.001 -ShowDetailsBrief
```

## Emulation — Test 1: Encoded Command

**Atomic test:** T1059.001-1  
**What it does:** Executes PowerShell with -EncodedCommand flag containing
a base64 encoded payload.

Run the atomic:

```powershell
Invoke-AtomicTest T1059.001 -TestNumbers 1
```

**Expected activity generated:**
- Sysmon EID 1: powershell.exe with `-EncodedCommand` in CommandLine
- Parent process: powershell.exe or cmd.exe

**Splunk validation query:**

```splunk
index=main sourcetype="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational"
EventID=1
Image="*\\powershell.exe"
CommandLine="*EncodedCommand*"
| table _time, ComputerName, User, CommandLine, ParentImage
```

**Result:** ✅ Detected — 1 event generated, query returned result within
30 seconds of execution.

## Emulation — Test 2: Hidden Window

**Atomic test:** T1059.001-4  
**What it does:** Launches PowerShell with -WindowStyle Hidden and
-NonInteractive flags — common in malware droppers.

```powershell
Invoke-AtomicTest T1059.001 -TestNumbers 4
```

**Splunk validation query:**

```splunk
index=main sourcetype="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational"
EventID=1
Image="*\\powershell.exe"
(CommandLine="*-WindowStyle Hidden*" OR CommandLine="*-NonInteractive*")
| table _time, ComputerName, User, CommandLine, ParentImage
```

**Result:** ✅ Detected — flags visible in CommandLine field.

---

## Emulation — Test 3: Download Cradle

**Atomic test:** T1059.001-13  
**What it does:** Uses PowerShell IEX + IWR (Invoke-WebRequest) to
download and execute a script from a remote URL — a classic living-off-the-land
technique used by APT29 and Lazarus.

```powershell
Invoke-AtomicTest T1059.001 -TestNumbers 13
```

**Splunk validation query:**

```splunk
index=main sourcetype="XmlWinEventLog:Microsoft-Windows-Sysmon/Operational"
EventID=1
Image="*\\powershell.exe"
(CommandLine="*IEX*" OR CommandLine="*Invoke-Expression*"
OR CommandLine="*IWR*" OR CommandLine="*Invoke-WebRequest*")
| table _time, ComputerName, User, CommandLine, ParentImage
```

**Result:** ✅ Detected — IEX and IWR visible in CommandLine.

## False Positives Observed

- Software deployment tools (SCCM) triggered the hidden window query
  during scheduled task execution. Mitigated by filtering ParentImage.

---

## Conclusion

Splunk detections for T1059.001 are effective against the most common
PowerShell execution variants used by APT29 and Lazarus Group.
Script Block Logging (EID 4104) was not enabled during this exercise —
enabling it would increase detection fidelity significantly.

**Recommendation:** Enable Script Block Logging via GPO to capture
full PowerShell script content regardless of obfuscation.
