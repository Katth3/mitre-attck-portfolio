# T1070.001 — Windows Event Log Cleared

**Tactic:** Defense Evasion  
**Technique:** T1070.001 — Indicator Removal: Clear Windows Event Logs  
**Data source:** Windows Security Log (EID 1102), Windows System Log (EID 104)  
**Threat actors:** APT29, Lazarus Group, ransomware operators  
**Confidence:** High — very few legitimate reasons to clear logs

---

## Query 1 — Security log cleared (EID 1102)

```splunk
index=main sourcetype="WinEventLog:Security"
EventCode=1102
| table _time, ComputerName, SubjectUserName, SubjectDomainName
| sort -_time
```

**What it detects:** Security audit log cleared. Event ID 1102 is generated
every time the Security event log is wiped. This is a high-fidelity indicator
of an attacker covering tracks post-compromise.

---

## Query 2 — System log cleared (EID 104)

```splunk
index=main sourcetype="WinEventLog:System"
EventCode=104
| table _time, ComputerName, UserData
| sort -_time
```

**What it detects:** System log cleared. Complements EID 1102 — attackers
often clear both logs to remove all traces of activity.

---

## Query 3 — Combined alert with context

```splunk
index=main (sourcetype="WinEventLog:Security" EventCode=1102)
OR (sourcetype="WinEventLog:System" EventCode=104)
| eval log_cleared_type=case(
    EventCode=1102, "Security Log Cleared",
    EventCode=104, "System Log Cleared")
| table _time, ComputerName, log_cleared_type, SubjectUserName
| sort -_time
```

**What it detects:** Both log clearing events in a single unified view.
Useful for correlation — if both fire within minutes, high probability
of active attacker on the system.

---

## Recommended alert threshold

Set as **real-time alert** — any single hit warrants immediate investigation.
This is not a noisy detection. False positive rate in production: very low.

---

## False positives

| Source | Description | Mitigation |
|--------|-------------|------------|
| IT operations | Intentional log clearing during maintenance | Require change ticket, whitelist by user |
| Forensic tools | Some tools clear logs during analysis | Only run forensics on isolated systems |
